# TO-DO LIST APP

## Project Overview

The To-Do List App is a simple and interactive task management application developed using HTML, CSS, and JavaScript. The main purpose of this project is to help users organize their daily activities and improve productivity by maintaining a list of tasks in a structured manner.

The application provides features such as creating tasks, editing task details, deleting unwanted tasks, marking tasks as completed, assigning priorities, and setting due dates. The app also uses browser local storage to save task data permanently, allowing users to access their tasks even after refreshing or reopening the application.

The interface is designed to be clean, modern, responsive, and user-friendly so that users can easily manage their tasks without difficulty.

---

# Objectives

* To create a simple and efficient task management application.
* To help users organize daily tasks effectively.
* To provide task tracking with completion status.
* To implement local data storage for persistence.
* To design a responsive and attractive user interface.

# Features of the Application

## 1. Add Task

Users can create new tasks by entering:

* Task title
* Description
* Priority level
* Due date

---

## 2. Edit Task

Existing tasks can be modified at any time. Users can update:

* Title
* Description
* Priority
* Due date

---

## 3. Delete Task

Tasks that are no longer needed can be removed from the list permanently.

---

## 4. Task Completion Status

Users can mark tasks as:

* Completed
* Active/Pending

Completed tasks are visually differentiated from active tasks.

---

## 5. Priority Management

Each task can be assigned a priority level:

* Low
* Medium
* High

This helps users focus on important tasks first.

---

## 6. Due Date Support

Users can assign deadlines for tasks, making time management easier.

---

## 7. Search and Filter

The application includes:

* Search functionality
* Task filtering
* Sorting options

This improves usability when handling multiple tasks.

---

## 8. Local Storage

All tasks are stored in browser local storage. Data remains available even after:

* Refreshing the page
* Closing and reopening the browser

---

# System Workflow

1. User opens the application.
2. User adds a new task.
3. Task details are stored in local storage.
4. User can:

   * Edit tasks
   * Delete tasks
   * Mark tasks completed
5. Updated data is automatically saved.
6. Tasks are displayed dynamically on the home screen.

# Limitations

## 1. No User Login System

The current version does not support multiple users.

---

## 2. Data Stored Only Locally

Tasks are stored only in browser local storage and cannot be accessed from another device.

---

## 3. No Real-Time Notifications

The app currently does not provide reminder alerts or notifications.

---

## 4. Limited Collaboration

Multiple users cannot share or collaborate on tasks.

---

## 5. Browser Dependency

If browser storage is cleared, all tasks will be lost.
